Categories of Web Applications & Architectures

1. Categories of Web Applications

Web applications can be categorized based on their main purpose and the type of services they provide to users.

Document-Centric Applications

  • Focus on creating, editing, and sharing documents online
  • Support real-time collaboration
  • Provide version control and editing tools
Examples: Google Docs, Microsoft Office Online

Social Web Applications

  • Designed for social interaction and communication
  • Allow users to share content, like posts, and comment
  • Include features such as profiles, feeds, and notifications
Examples: Facebook, Twitter, Instagram, LinkedIn

Semantic Web Applications

  • Use structured data to give meaning to information
  • Improve search accuracy and data integration
  • Enable systems to understand relationships between data

Key Concept: Interoperability – the ability of different systems to communicate and exchange data effectively.

Examples: Wolfram Alpha, Google Knowledge Graph

2. Web Architectures

Web architecture defines how different components of a web application are organized and interact with each other.

Layered Architecture

  • Divides the application into multiple layers
  • Common layers include Presentation, Business Logic, and Data
  • Improves modularity and scalability

One-Tier Architecture

  • All components are combined into a single system
  • Presentation, logic, and data exist together
Example: Simple desktop applications

Two-Tier (Client-Server) Architecture

  • The client manages the user interface
  • The server handles data storage and processing
Example: Basic online form systems

Three-Tier Architecture

  • Presentation Layer – User interface
  • Application Layer – Business logic
  • Data Layer – Database management
  • Provides better scalability and maintainability

N-Layered Architecture

  • Extension of three-tier architecture
  • Adds more specialized layers such as Domain and Data Access
  • Offers high flexibility and modular development

Key Insight: Choosing the right architecture helps developers build scalable, maintainable, and efficient web applications.
