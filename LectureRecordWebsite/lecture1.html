Lecture 1

Introduction to Web Technologies

1. Web Platforms Defined

Web platforms are digital environments or frameworks used for building and deploying web applications. They allow developers and users to interact, develop services, and deliver content over the internet.

Key Components

  • Infrastructure (servers, databases, networks)
  • Development tools (HTML, CSS, JavaScript, frameworks)
  • Content Management Systems
  • User Interface elements
  • APIs for system integration
  • Security features
Examples: Facebook, Shopify, WordPress, AWS, Netflix

2. HTTP Protocol

HTTP (HyperText Transfer Protocol) is the foundation of web communication. It works on a request-response model between the client (browser) and the server.

HTTP Methods

  • GET
  • POST
  • PUT / PATCH
  • DELETE

Status Codes

  • 1xx – Informational
  • 2xx – Success
  • 3xx – Redirection
  • 4xx – Client Error
  • 5xx – Server Error

HTTPS: Secure version of HTTP using SSL/TLS encryption.

3. Web Generations & Standards

Key Standards Organizations

  • W3C
  • IETF
  • WHATWG

Major Standards

  • HTML5 – Latest markup language
  • CSS3 – Includes Flexbox and Grid
  • JavaScript / ECMAScript – ES2016, ES2018, ES2020, ES2025
  • WCAG – Web Content Accessibility Guidelines
  • OWASP – Security best practices
  • Performance tools such as Google Lighthouse

4. Web Constraints

Technical

  • Browser and device compatibility
  • Bandwidth limitations

Design

  • User Experience (UX)
  • Accessibility
  • Consistency

Legal / Ethical

  • GDPR
  • CCPA
  • Copyright
  • Dark patterns

Content

  • Localization
  • SEO
  • Multimedia management

Key Insight: Successful web development requires balancing standards with constraints to create functional, accessible, and high-performance applications.
